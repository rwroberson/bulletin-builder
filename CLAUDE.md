# Bulletin App — Project Context for Claude Code

This file provides full context for building a desktop bulletin-generation app
for a small confessional Presbyterian church. Read this before doing anything else.

---

## Who This Is For

The primary user is Reid Roberson, a solo pastor at a small confessional
Presbyterian church (EPC). He has a background in mathematics, an M.Div., and
is proficient in LaTeX, shell scripting, Greek, Hebrew, and web development.

The secondary users are church secretaries and future pastors/deacons with no
assumed technical knowledge. The app must be intuitive enough for a non-technical
church secretary to use without training.

The longer-term vision is an open-source tool other small churches could adopt.
Start by solving Reid's specific workflow; generalize later.

---

## The Problem Being Solved

Reid produces a weekly church bulletin as a PDF. His current workflow involves:

1. Editing a `schedule.csv` file with the week's readings, hymns, and other
   liturgical data
2. Running `build-order.sh` to generate a TSV from the CSV
3. Running `tex-order.sh` (which calls `order.awk`) to convert the TSV to LaTeX
4. Separately editing `announcements.txt`, `prayer-list.txt`, `birthdays.txt`
5. Running `pdflatex` on `BULLETIN.tex` (which `\input`s all the above)
6. Running `pdflatex` on `book.tex` to produce an imposition for printing

This workflow is powerful but requires command-line knowledge. The app replaces
steps 1–6 with a friendly GUI while keeping the file structure intact.

---

## Design Philosophy

### Workspace Model (like Obsidian / VSCode)
The app opens a folder as a "vault." All data stays in plain, human-readable
files. The app reads and writes those files — it does not maintain a proprietary
database. A user who loses access to the app still has all their data.

### Two-Mode UI
- **Secretary mode**: A clean form-based interface. Date, readings, hymns,
  announcements. No markup, no file paths, no compilation errors visible.
- **Power-user mode**: Access to underlying templates, raw file editing, and
  advanced configuration. Reid will use this; a secretary never needs to see it.

### Plain Files Always Win
Never store data in a format that requires the app to read. `schedule.csv`,
`csh.tsv`, `announcements.txt`, etc. remain plain text files editable in any
text editor. The app is a convenience layer, not a lock-in mechanism.

## Key Files Explained

### schedule.csv
One row per Sunday. Columns:
```
DATE, Season, Introit, Call to Worship, Opening Hymn, Law Reading,
Assurance, Confession Hymn, OT Reading, Scripture Hymn, Sermon Text,
Closing Hymn, Notes
```
Hymn fields use codes looked up in `csh.tsv`:
- Plain numbers (`341`) = hymn in the 1948 Church Service Hymnal
- `P` prefix (`P103`) = psalm in the Trinity Psalter
- `TPH` prefix (`TPH325`) = hymn in the Trinity Psalter Hymnal
- `BH` prefix = Baptist Hymnal
- Text not in csh.tsv is printed as-is

Notes column flags: `Communion`, `Presbytery`, `V` (vacation), etc.

### csh.tsv
Tab-separated: `NUM`, `TITLE`, `TUNE`
Used for hymn autocomplete and for generating the formatted hymn line in the
order of worship. `build-order.sh` calls `hymnfo()` to look up hymns here.

### build-order.sh
Key flags:
- `-c` adds the Lord's Supper section to the order
- `-d "YYYY-MM-DD"` selects a specific date (defaults to next Sunday)

Outputs a TSV to stdout. Each row is one element of the order of worship.
Special rows:
- `HD\tSection Title` = a section heading (e.g. "God Calls Us")
- `\input{...}` rows = raw LaTeX passthrough
- Standard rows = `Element Name \t Scripture Reference \t Right-side note`

### announcements.txt
Uses LaTeX `description` environment for bullet items:
```latex
\subsection*{Announcements}
\begin{description}
  \item [Label] Body text.
\end{description}
```
Also uses custom `\announcec{Title}{Content}` and `\announcel{Title}{Content}`
macros for boxed announcements (centered and left-aligned respectively).

### BULLETIN.tex
Half-sheet format (5.5in × 8.5in, 0.25in margins, 12pt).
Key custom macros:
- `\bulletinline{Element}{Scripture}{Right note}` — standard order row
- `\bulletinhead{Section}` — decorative section heading
- `\responseall{text}` / `\responsemin{text}` — congregational response formatting
- `\announcec{Title}{Content}` / `\announcel{Title}{Content}` — boxed announcements
- `\sermscript{...}` — sermon info block

Uses `datenumber` package to auto-calculate next Sunday's date.
Font: Coelacanth (nf variant). Encoded T1.

### book.tex
Imposes the bulletin PDF as a 2-up booklet (signature of 4, landscape, angle 180).
Uses `pdfpages` package.

## V0 Scope (Build This First)

The minimum viable app that solves Reid's immediate problem:

1. **Open workspace** — pick the `~/doc/church/bulletin/` folder
2. **Weekly bulletin form** — load/edit the current week's row in `schedule.csv`:
   - Date picker (defaults to next Sunday)
   - Fields for all schedule.csv columns
   - Hymn fields with autocomplete from `csh.tsv` (type number, see title/tune)
   - Communion checkbox (maps to `-c` flag in build-order.sh)
3. **Announcements editor** — structured editor for `announcements.txt`:
   - Add/remove/reorder `\item` entries with label + body fields
   - Add boxed announcements (`\announcec` / `\announcel`)
   - Separate fields for prayer list and birthday list
4. **Build** — button that runs the full pipeline:
   - `build-order.sh` → `tex-order.sh` → `pdflatex BULLETIN.tex` → `pdflatex book.tex`
   - Creates the `YYYY-MM-DD/` folder if it doesn't exist
   - Shows stdout/stderr in a collapsible log panel
5. **Preview** — display the generated PDF inline after a successful build

Do not build formatting options, template editors, image managers, or multi-church
support yet. Those come later.

## Tech Stack

- **Framework**: Electron (cross-platform desktop, local file access, can shell out)
- **PDF generation**: `pdflatex` (must be installed on the system; not bundled)
- **Frontend**: Keep it simple — plain HTML/CSS/JS or lightweight React.
  Match the feel of iA Writer or Obsidian: calm, typographic, focused.
- **Data**: Plain files only. No SQLite, no proprietary database for v0.
- **Shell integration**: Node `child_process` to run existing bash scripts

### Future stack considerations (not for v0)
- Puppeteer/Chromium for HTML→PDF if we want to remove the LaTeX dependency
  for non-technical users at other churches
- SQLite for schedule/hymnal data once multi-church use is real
- Shared community hymnal library (online, opt-in)

---

## UI Principles

- **Default to simple.** The form should feel like filling out a church bulletin
  template, not configuring software.
- **Autocomplete hymns.** Hymn number fields should look up and display the
  title and tune name from csh.tsv as the user types.
- **Preserve the files.** Every save writes to the actual plain-text files.
  Never abstract the user away from their own data.
- **Errors should be human-readable.** If pdflatex fails, show the relevant
  error line, not the full log dump.
- **No onboarding friction.** Open a folder, start working. No account creation,
  no license keys, no telemetry.

## What NOT to Build Yet

- Multi-template support
- Image manager
- Multiple bulletin formats/sizes
- Multi-church / cloud sync
- Installer / auto-updater
- Anything requiring a network connection

## Conventions

- All dates in `YYYY-MM-DD` format
- Hymn codes follow the `csh.tsv` prefix conventions (plain, P, TPH, BH)
- LaTeX files use UTF-8 encoding
- Shell scripts assume bash on Ubuntu/Debian Linux
- The app runs on the same machine as the LaTeX installation (Reid's Linux server
  or local machine)

## Context on Liturgical Structure

The order of worship follows a fourfold pattern common in Reformed liturgy:

1. **God Calls Us** — Prelude, Welcome, Call to Worship, Invocation, Opening Hymn
2. **God Cleanses Us** — Law, Confession, Assurance, Hymn
3. **God Consecrates Us** — OT Reading, Children's Word, Hymn, NT Reading, Sermon,
   Intercession, Creed, Gloria Patri, Offertory, Doxology
4. **[God Communes with Us]** — Lord's Supper (when observed, ~monthly)
5. **God Commissions Us** — Closing Hymn, Benediction, Departure Hymn

The Benediction is always Numbers 6:24-26. The Departure Hymn is always
"Shalom to You" (SOMOS DEL SENOR). The Gloria Patri is always #436 (GREATOREX).
The Doxology is always #434 (OLD 100TH). These are fixed and should not be
editable fields in v0.

The confession prayer and call to worship text are drawn from files in
`elements/`. In v0, these can be plain text fields or dropdowns pointing to
the elements folder. Full responsive-reading editors come later.
